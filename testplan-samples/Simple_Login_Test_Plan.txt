LOGIN SYSTEM TEST PLAN

Test Case 1: Valid Login
Objective: Verify users can login with valid credentials

Prerequisites:
- System is running
- User account exists with username "testuser" and password "password123"

Test Steps:
1. Navigate to login page
   Expected: Login form displays with username and password fields

2. Enter valid username "testuser"
   Expected: Username field accepts input

3. Enter valid password "password123"
   Expected: Password field accepts input and masks characters

4. Click Login button
   Expected: User is redirected to dashboard page

5. Verify user session is active
   Expected: User name displays in header, logout option available

Priority: High
Type: Functional

Test Case 2: Invalid Password
Objective: Verify system rejects invalid passwords

Prerequisites:
- System is running
- Valid username "testuser" exists

Test Steps:
1. Navigate to login page
   Expected: Login form displays

2. Enter valid username "testuser"
   Expected: Username field accepts input

3. Enter invalid password "wrongpassword"
   Expected: Password field accepts input

4. Click Login button
   Expected: Error message "Invalid credentials" displays

5. Verify user remains on login page
   Expected: User is not logged in, stays on login form

Priority: High
Type: Security

Test Case 3: Empty Fields Validation
Objective: Verify required field validation

Test Steps:
1. Navigate to login page
2. Leave username field empty
3. Leave password field empty
4. Click Login button
   Expected: Validation errors display for both fields
5. Enter username only, leave password empty
6. Click Login button
   Expected: Password required error displays
7. Enter password only, leave username empty
8. Click Login button
   Expected: Username required error displays

Priority: Medium
Type: Validation
